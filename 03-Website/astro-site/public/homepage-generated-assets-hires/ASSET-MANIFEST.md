## Notes

- Load `website-reference-mockup.png` first during implementation and use it as the visual source of truth.
- This is the correct source folder for building the homepage.
- The assets are isolated and clean, with no surrounding webpage chrome.
- Most assets contain no text. The exception is `ebook-cover.png`, where the cover typography is part of the asset itself.
- The earlier extracted pack remains available in `assets/homepage-mockup-assets` as a visual reference set, but it should not be used as the implementation source when the generated version exists here.
