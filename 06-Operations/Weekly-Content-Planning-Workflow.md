# Weekly Content Planning Workflow

Every Friday, 60–90 minutes. The ritual that keeps the 30-day plan responsive instead of mechanical.

## When

**Fridays, 4–6 PM** (per [[KPIs-Tracking]]). End of the production week, before the weekend, with one workday's slack to react.

## Inputs

- Pinterest Analytics (last 7 days).
- Affiliate dashboards (Etsy / Awin, Wayfair / CJ, CB2 / Impact, 54kibo / ShareASale).
- [[Pin-Production-Queue]] — what shipped, what's blocked.
- [[30-Day-Pinterest-Content-Plan]] — what's coming.
- [[KPIs-Tracking]] — last week's snapshot.

## The 6 steps

### 1. Did this week's plan ship? (10 min)

- Count `[✓]` vs `[ ]` in [[Pin-Production-Queue]].
- For any `[ ]` that should have shipped: why? Move to `[!]` blocked with reason.
- For any `[!]`: name the blocker, owner, unblock date. Push to next week if not resolvable.

### 2. Pinterest Analytics review (15 min)

In Pinterest Analytics, sort the last 7 days' pins by:

- **Outbound clicks** — top 3, bottom 3.
- **Saves** — top 3.
- **Impressions** — top 3.

Note in [[KPIs-Tracking]]:
- Total impressions (target: rising week-over-week).
- Total saves.
- Total outbound clicks.
- Average CTR.
- Top 3 pins (ID + saves + outbound clicks).
- Bottom 3 pins (ID + impressions — they may be brand-new and need more time, or genuinely dead).

### 3. Pattern recognition (10 min)

For the top 3 pins, what do they share?
- World (Neo Deco / Afrohemian / Adire / Brass / Trends)?
- Format (Roundup / How-To / Mistakes / Trend Forecast)?
- Headline pattern (numbered, "Stop X," "How to," etc.)?
- Color palette dominance?
- Time of day posted?

For the bottom 3, the same question — and add: did the title and image agree? Did the destination URL load?

Write 1–3 sentences in [[KPIs-Tracking]] under "This week's pattern."

### 4. Refresh top performers (15 min)

For each of the top-3, **produce 2 fresh variations** for next week:
- New image, same destination URL.
- Different headline formula from [[Headline-Formulas]] targeting the same keyword.
- Different Canva template.

Add the 6 fresh-variation pins to next week's [[Pin-Production-Queue]] as priority slots.

### 5. Affiliate review (10 min)

In each affiliate dashboard:
- Which products got clicks but **zero sales**? Flag for swap (cheaper alternative or better-reviewed product per [[Product-System]]).
- Which products had ≥1 sale? Note for "winning products" list — feature in next week's content.
- Any "out of stock" or "discontinued" products in current pins? Pull or swap.
- Total affiliate revenue this week (note in [[KPIs-Tracking]]).

### 6. Update next week's plan (10 min)

In [[Next-7-Days-Priority-Plan]]:
- Refresh the "What ships this week" table with the upcoming 7 days.
- Add the 6 fresh-variation pins from step 4.
- Confirm blog destination URLs for next week's pins.
- Note affiliate priorities.
- Note cultural-care audits required.
- Note risks.

## Output of the weekly review

By 6 PM Friday:
- [[KPIs-Tracking]] updated with this week's snapshot.
- [[Pin-Production-Queue]] updated for next week (next 10 pins clear).
- [[Next-7-Days-Priority-Plan]] re-written for the new week.
- 6 fresh-variation pins queued.
- 1–3 affiliate products swapped if needed.

## Monthly variant (every 4th Friday)

Add 60 minutes to:
- Run the **30-day retro** in [[KPIs-Tracking]] (compare to day-30 / day-60 / day-90 benchmarks).
- Re-evaluate world distribution (is one world over-indexing? per [[KPIs-Tracking]] day-90 protocol).
- Generate the next 30-day pin plan using [[AI-Prompts]] §7.13.
- Update [[Existing-Board-Audit]] with current pin counts and any board health changes.
- Re-validate keywords against current Pinterest Trends data.

## Anti-patterns

- **Reviewing without acting.** A weekly review that produces no pin-queue changes is a meeting, not a workflow.
- **Reacting to a single pin's spike.** One viral pin doesn't justify replanning the month. Wait for two consecutive weekly winners in the same world before shifting strategy.
- **Skipping the affiliate review.** That's where the revenue actually is.
- **Adding new pin concepts without retiring under-performers.** The queue should stay at ~10 — top up, don't pile.
